## RULES
1. Output ONLY valid JSON — no markdown, no explanation, no code fences
2. overall_status must be:
   - "PASSED" if ALL steps passed
   - "FAILED" if ANY step failed and could not be healed
   - "PARTIALLY_PASSED" if some steps failed but others passed (including healed ones)
3. Include actionable recommendations based on the failures observed
4. For unresolved failures, always suggest a specific manual fix
5. Keep notes concise but informative
6. Do NOT fabricate results — only report what actually happened
